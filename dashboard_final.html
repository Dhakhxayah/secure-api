💣 DoS Body Size Test
How It Works
The server has RequestBodyLimitLayer set to 1MB (1,048,576 bytes).
Any request body larger than 1MB is rejected with 413 before even being parsed.
This prevents memory exhaustion and JSON parsing attacks.

All queries filter WHERE deleted_at IS NULL — deleted user becomes invisible.

🔒 AES-256-GCM Encryption Test
How It Works
Email stored as: base64(nonce):base64(ciphertext) — unreadable without the key.
SHA-256 hash stored separately for WHERE clause lookups.
Server decrypts on every read — you see plain email, DB has ciphertext.

⚠ Use a FRESH email not used before. If you get "Load failed" run redis-cli FLUSHDB first.

Verify Ciphertext in PostgreSQL
After running the test, paste this into your terminal to see the raw encrypted data:

psql -U api_user -d secure_api -c "SELECT email, email_hash FROM users ORDER BY created_at DESC LIMIT 1;"

The email column will show: base64nonce:base64ciphertext — completely unreadable without the encryption key.
